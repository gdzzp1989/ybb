~~js跨域：指不同的域之间进行数据传输或通信，只要协议（http和https）、子域名、端口有任何一个不同，都是不同的域。
  解决：
    跨域资源共享（CORS）：在被请求的Response header加入以下设置
		'Access-Control-Allow-Origin:*'//指定允许其他域名访问		
		'Access-Control-Allow-Methods:GET,POST'//响应类型		
		'Access-Control-Allow-Headers:x-requested-with,content-type'//响应头设置

    jsonp（JSON with Padding填充式json）: 只能GET
                 传统js：利用script标签的src="XXX?callback=XX"属性进行跨域，另一个域再用callback参数上的XX方法名回调自己的函数获取值。
		 Jquery写法1：$.getJSON('http://example.com/data.php?callback=?,function(jsondata)'){
					//处理获得的json数据
				});
                 Jquery写法2：ajax 使用参数 dataType:'jsonp'

    window.postMessage(message,targetOrigin)（HTML5）：用postMessage方法来跨域传送数据

    document.domain:设置主域值，可以通过js访问到iframe中的属性和对象了（不适用ajax的方法）

    window.name：利用iframe的src得到其他域设置的window.name属性，再修改src同域获取window.name


~~cookie跨域访问：
  解决顶级域名与二级域名之间的跨域问题
    cookie的两个属性:
       domain（域）：domain属性的默认值是创建cookie的服务器的主机名。通过设置这个属性可以使多个web服务器共享cookie。
       path（路径）：表示创建该cookie的服务器的哪些路径下的文件有权限读取该 cookie,默认为/，就是创建该cookie的服务器的根目录。
  解决顶级域名于顶级域名之间的跨域问题
   nginx反向代理
   jsonp
      Jquery：ajax 使用参数 dataType:'jsonp'
   CORS跨域传送cookie
     Jquery使用ajax写法 crossDomain: true,
			xhrFields: {
                          withCredentials: true
			},

  $("*")	所有元素
  $("#lastname")	id="lastname" 的元素
  $(".intro")	所有 class="intro" 的元素
  $("p:first")	第一个 <p> 元素
  $("ul li:eq(3)")	列表中的第四个元素（index 从 0 开始）
  $("[href='#']")	所有 href 属性的值等于 "#" 的元素
  $("[href$='.jpg']")	所有 href 属性的值包含以 ".jpg" 结尾的元素
  $(":input")	所有 <input> 元素
  $(":text")	所有 type="text" 的 <input> 元素
  $(":submit")	所有 type="submit" 的 <input> 元素
